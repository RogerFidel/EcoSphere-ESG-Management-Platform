# 🚀 GreenQuest Deployment & Configuration Guide

This document outlines instructions for deploying the **GreenQuest ESG Gamification Platform** to production environments using Docker Compose and setting up CI/CD pipelines.

---

## 🛠️ Environment Variables Configuration

GreenQuest uses environment files configuration. Create a `.env` file inside `backend/` by referencing `backend/.env.example`.

### Core Variables

| Variable | Description | Recommended Production Value |
| :--- | :--- | :--- |
| `PORT` | API Server listening port | `5000` |
| `NODE_ENV` | Runtime environment | `production` |
| `JWT_SECRET` | Secret key used for signing JWTs | *Generate secure 32+ character string* |
| `OPENAI_API_KEY` | OpenAI API access key | *Your premium OpenAI key* |
| `DB_HOST` | Database server hostname | `postgres` (or remote hostname) |
| `DB_PORT` | Database server port | `5432` |
| `DB_USER` | Database connection username | `greenquest` |
| `DB_PASS` | Database connection password | *Secure password string* |
| `DB_NAME` | Database schema name | `greenquest` |
| `REDIS_URL` | Redis URL | `redis://redis:6379` |

## 🐳 Containerized Production Deployment

GreenQuest is fully containerized. A multi-stage production deployment builds the optimized React static build, serves it via Nginx, and connects the API backend to database layers.

### Steps to Run

1.  **Configure Environment Variables:**
    Copy and adjust environments:
    ```bash
    cp backend/.env.example backend/.env
    # Edit backend/.env and insert your database credentials and OPENAI_API_KEY
    ```

2.  **Deploy Stack via Docker Compose:**
    Launch all services (Nginx frontend, API backend, Postgres database, Redis cache) in background mode:
    ```bash
    docker compose up -d --build
    ```

3.  **Execute Migrations & Seeds inside Container:**
    Build database tables and run mock seed script:
    ```bash
    docker compose exec backend npm run seed
    ```

4.  **Verification:**
    Check container health statuses:
    ```bash
    docker compose ps
    ```
    *   **Frontend Webapp:** accessible at `http://localhost` (port 80)
    *   **Backend REST API:** accessible at `http://localhost:5000/api/v1`
    *   **API Healthcheck:** verification endpoint at `http://localhost:5000/health`

## 🔄 Local Dev Infrastructure Mode

If you prefer to run Node and React processes natively on your machine, you can run only PostgreSQL and Redis using dev compose:

```bash
# Spin up infrastructure
docker compose -f docker-compose.dev.yml up -d

# Start backend natively
cd backend
npm run dev

# Start frontend natively
cd ../frontend
npm run dev
```

## 🚀 CI/CD Pipeline Integration

GreenQuest includes a robust GitHub Actions workflow inside `.github/workflows/ci-cd.yml`.

### Pipeline Jobs
1.  **Backend Tests:** Launches containerized Postgres and Redis services, executes Jest unit and integration suites, and uploads coverage summaries.
2.  **Frontend Build:** Verifies that Vite compiles the production bundles.
3.  **Security Audit:** Analyzes dependencies packages for known vulnerabilities.
4.  **Docker Build & Push:** Builds both backend and frontend images, tagging them with `latest` and `sha-` commits, then pushes to **GitHub Container Registry (GHCR)**.

### Secret Settings Required
To activate full pipeline capabilities, define the following secrets in GitHub Repository Settings:
*   `GITHUB_TOKEN`: Automatically generated by GitHub for package registries access.
*   `CODECOV_TOKEN`: *Optional* token for code coverage tracking integrations.
*   `OPENAI_API_KEY`: *Optional* if integration tests call OpenAI APIs (fully mocked in current suites).

## 🧹 Maintenance and Monitoring

### logs tracking
Winston logs queries and records daily logs in `backend/logs/combined.log` and error logs in `backend/logs/error.log`.
To view live container logs:
```bash
docker compose logs -f backend
```

### Database Backups
To create a dump of the PostgreSQL production database:
```bash
docker compose exec postgres pg_dump -U greenquest greenquest > greenquest_backup.sql
```
To restore a backup file:
```bash
docker compose exec -T postgres psql -U greenquest greenquest < greenquest_backup.sql
```
